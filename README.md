# Practical lesson pz-UML
Побудова повердінкових UML-діаграм для проєктування інформаційних систем

## Тема - Програмний модуль генерації оптимальних логістичних маршрутів із використанням методу аналізу ієрархій

## 1. Діаграма варіантів використання (Use Case Diagram)

Ця діаграма описує функціональність системи з точки зору користувача (Логіста) та зовнішніх систем.

* **Актори:** Логіст (основний) та Maps API (допоміжний для отримання геоданих).
* **Ключові сценарії:** Налаштування критеріїв (час, вартість, безпека) є критичним для МАІ. Генерація маршруту включає звернення до зовнішніх сервісів карт.

`code`

    graph LR
    User((Логіст))
    SystemBoundary[Модуль генерації маршрутів]
    
    subgraph SystemBoundary
        UC1(Введення параметрів маршруту)
        UC2(Налаштування критеріїв МАІ)
        UC3(Розрахунок ваг критеріїв)
        UC4(Генерація оптимального маршруту)
        UC5(Перегляд результатів на карті)
        UC6(Експорт звіту)
    end
    
    MapsAPI((External Maps API))

    User --> UC1
    User --> UC2
    User --> UC4
    UC2 ..> UC3 : <<include>>
    UC4 --> MapsAPI
    UC4 ..> UC5 : <<extend>>
    User --> UC6

`code`

## 2. Діаграма послідовності (Sequence Diagram)

Відображає процес взаємодії об'єктів під час виконання основного завдання: розрахунку маршруту.

Ця діаграма фокусується на "спілкуванні" між компонентами. Вона демонструє, як пріоритети користувача проходять крізь математичний двигун МАІ перед тим, як система звернеться до картографічних даних.

`code`

    sequenceDiagram
    participant L as Логіст
    participant UI as Інтерфейс користувача
    participant C as Контролер модуля
    participant AHP as Двигун МАІ (AHP Engine)
    participant MAP as Maps Service API

    L->>UI: Вводить дані точок та пріоритети критеріїв
    UI->>C: Передає вхідні дані
    
    rect rgb(240, 240, 240)
        Note over C, AHP: Процес аналізу ієрархій
        C->>AHP: Передає матрицю попарних порівнянь
        AHP->>AHP: Розрахунок власних векторів та Індексу узгодженості
        AHP-->>C: Повертає вагові коефіцієнти критеріїв
    end

    C->>MAP: Запит можливих маршрутів та відстаней
    MAP-->>C: Масив альтернативних маршрутів
    
    C->>AHP: Розрахунок інтегральної оцінки альтернатив
    AHP-->>C: Оптимальний варіант (Ranked List)
    
    C-->>UI: Відображення кращого маршруту
    UI-->>L: Результат генерації
    
`code`

## 3. Діаграма діяльності (Activity Diagram)

Описує внутрішню логіку роботи модуля, зокрема ітераційний процес перевірки узгодженості матриць у МАІ.

Діаграма діяльності ілюструє важливий аспект МАІ — перевірку на логічну узгодженість думок експерта. Якщо користувач вказав суперечливі пріоритети, система змушує повернутися до етапу заповнення матриці.

`code`

    stateDiagram-v2
    [*] --> Start: Початок формування маршруту
    Start --> InputData: Введення пунктів призначення та критеріїв
    
    state "Обробка МАІ" as AHP_Process {
        InputData --> MatrixCreation: Створення матриці попарних порівнянь
        MatrixCreation --> CalcWeights: Розрахунок ваг та Індексу узгодженості (CI)
        
        Decision_CI: CI < 0.10?
        CalcWeights --> Decision_CI
        
        Decision_CI --> MatrixCreation: Ні (Коригування значень)
        Decision_CI --> FinalWeights: Так (Дані узгоджені)
    }
    
    FinalWeights --> GetRoutes: Отримання варіантів маршрутів з БД/API
    GetRoutes --> Scoring: Розрахунок рейтингу кожної альтернативи
    Scoring --> BestRoute: Вибір маршруту з найвищим балом
    BestRoute --> Display: Відображення на екрані
    Display --> [*]
    
`code`

### Опис роботи

Даний проєкт реалізує інтелектуальну надбудову над стандартними логістичними системами. Використання Методу аналізу ієрархій дозволяє враховувати не лише найкоротшу відстань, а й суб'єктивні чинники, важливі для конкретного бізнесу:

* Діаграма **Use Case** визначає межі системи та роль логіста у процесі прийняття рішень.

* Діаграма **Sequence** деталізує технічну реалізацію, показуючи розділення логіки інтерфейсу та математичних розрахунків.

* Діаграма **Activity** візуалізує алгоритмічний цикл, що забезпечує математичну точність вибору маршруту.
